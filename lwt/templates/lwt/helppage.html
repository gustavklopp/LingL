{% trans "Creating your own dictionary link:" %}

{% blocktrans %}

LingL provides a few choice of dictionary links which could work for your language (like Wiktionary, Google etc...)

Those dictionary links are of the form: '!https://....'. The '!' at the start means that LingL has an already defined way to fetch this dictionary website (i.e an "API") and a way to click on the translation (or use shortcut) to automatically set as a translation.

But you are not restricted to those dictionaries and you can provide your own link to your prefered dictionary.

The easiest way is to look at this predefined dictionary links to use a example. But here how it works:

  • 1. Try to do a simple search for a word on the online dictionary that you want in your browser:
    The URL of the word 'test' for example may look like: "https://en.pons.com/translate/english-french/test"
  • 2. Replace 'test' with '<WORD>' : '<WORD>' will be processed as the searched word. so, for example: write: https://en.pons.com/translate/french-english/<WORD> (NB: it's only an example since this dictionary is already amongst the built-in LingL dictionary links)
  • 3. Warning! Some dictionary webpages prevent the opening of their website on LingL (or any other third party websites) (for ex. Google translate).

    When trying to look out a word, you will see an error message like: a security error prevented the resource from being loaded.

    LingL has 2 ways to counteract this:

    • 3.1 Make the website opening in a new window: To have it, prepend the link with the '*' character. For ex.: write *https://en.pons.com/translate/french-english/###

    • 3.2 Make LingL scrape the website from its content: To have it prepend the link with the '^' character. For ex.: write ^https://translate.google.com/?sl=fr&tl=en&text=test&op=translate

      The first solution may make the website faster than the second one but with the disadvantage of a new window. The scraped website may also lead to wrong layout of the original dictionary website.

    • P.S: You can also ask on LingLibre if anybody would like to create an API for your particular dictionary.
